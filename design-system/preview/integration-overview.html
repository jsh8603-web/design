파이프라인 책임 분담

클라우드(이 프로젝트) = 룰북 · 로컬(slides-grab) = 공장

클라우드 (이 프로젝트)

DESIGN SYSTEM SOURCE OF TRUTH
  • 규칙·토큰·레이아웃·프롬프팅 룰 박제
  • LLM 이 슬라이드 HTML 생성 (image-slot 마커 포함)
  • 디자인 리뷰 + image_slot.js drag-drop 시연
  • gen_pptx 로 변환 sanity check
  • 실패 모드 발견 시 룰 파일 갱신
⇄ SYNC

로컬 (slides-grab 리포)

RUNTIME + SELF-LEARNING
  • generate-images.mjs — Gemini 이미지 생성, 슬롯 swap
  • preflight-html.js — PF rules 자동 검증
  • html2pptx.cjs — Playwright + PptxGenJS final 변환
  • QA 9-check + autofix · vqa-batch.mjs Vision Validation
  • pptx-inspection-log.md 누적 (51KB), audit/ 기록

한 줄 mental model: 클라우드는 룰북, 로컬은 공장. 룰북이 바뀌면 공장이 따른다. 공장에서 새 실패를 발견하면 룰북이 갱신된다. 자세한 sync 절차: pipeline_handoff.md.

슬라이드 생성 의무 순서 (prompting_rules.md §0): ① 테마 결정 → ② 레이아웃 카탈로그 선택 → ③ 매트릭스 검증 (✅) → ④ LayoutSchema 준수 → ⑤ invariants 적용 → ⑥ PF rules 준수 → ⑦ QA 자가 검토.

v3 신규 PF 룰: PF-69 (매트릭스 위반) · PF-70 (data-layout 의무) · PF-71 (data-theme 의무) · PF-72 (LayoutSchema 위반) · PF-73 (primary 풀블리드 금지) · PF-74 (whitespace) · PF-75 (명사형 종결).
