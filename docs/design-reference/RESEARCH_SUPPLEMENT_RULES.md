# 리서치 보완 규칙 (Research Supplement Rules)

소스 자료 부족 시 search-engine 스킬(Gemini 2-Phase)로 보완하는 절차.
훅/가드 없이 **기존 의무 절차(아웃라인 템플릿, 수정 피드백 형식, progress 체크박스)에 녹여서** 실효성을 확보한다.

---

## §1. 소스 부족 판정 — 구조적 기준 (별도 계산 없이 즉시 판단)

아웃라인 작성 중 아래 3가지를 자연스럽게 확인한다. **하나라도 해당하면 보완 리서치 트리거.**

| 기준 | 부족 신호 | 판정 방법 |
|------|----------|----------|
| **토픽 커버리지** | 아웃라인 섹션에 대응하는 소스 내용이 없음 | 아웃라인 `Source:` 필드를 채울 때 빈 칸이 있으면 부족 |
| **구체성 테스트** | 소스에서 수치/사례/고유명사를 3개+ 인용할 수 없음 (섹션당) | Details 작성 시 구체적 팩트 없이 일반론만 나오면 부족 |
| **NLM 출력 분량** | NLM/링크 결과가 체감 2페이지 미만 | 한눈에 판단 |

---

## §2. 아웃라인 `Source:` 필드 (plan-skill 연동)

slide-outline.md의 각 슬라이드 섹션에 `Source:` 필드를 필수로 포함한다.
이 필드를 채우는 행위 자체가 소스 검증 역할을 한다.

```markdown
### Slide 3 - 시장 규모 현황
- **Type**: Content
- **Key Message**: 글로벌 시장 2025년 $4.2B
- **Details**:
  - 글로벌 시장 2025년 $4.2B (CAGR 12.3%)
  - 국내 시장 점유율 상위 3사
- **Source**: NLM p.3 "Global Market Report 2025" / search-engine R-01
- NanoBanana: ...
```

**규칙**:
- `Source:` 가 비어있거나 `(없음)` → 해당 토픽 보완 리서치 필수
- AI 일반 지식만으로 작성한 내용은 `Source: AI general knowledge` 로 표기 — 사용자가 소스 기반 여부를 구분 가능
- 리서치 보완 후 `Source:` 에 `search-engine R-{NN}` 추가

## §3. progress.md 체크박스 (기존 checklist-guard 연동)

Step 1 체크박스에 소스 커버리지 항목을 포함한다. checklist-guard가 기존대로 감시.

```markdown
## 완료 단계
- [x] Step 0 소스 확인
- [ ] Step 0.5 소스 커버리지: 모든 섹션 Source 필드 기입 완료, 빈 Source 없음
- [ ] Step 1 아웃라인 승인
```

---

## §4. 사용자 보완 요청 → 수정 피드백 형식에 리서치 통합

아웃라인(Step 1) / 초안(Step 1.5A) / 수정(Step 4) 단계에서 사용자가 **내용 보완**을 요청하면, 기존 수정 피드백 번호 리스트에 리서치 행을 첫 번째로 삽입한다.

```
수정 내용:
1. [리서치] 슬라이드 5 "시장 규모" — search-engine 스킬로 최신 데이터 확보
2. 슬라이드 5: 시장 규모 $3.8B → $4.2B (2025 기준) 업데이트
3. 슬라이드 5: Source 필드에 리서치 출처 추가

진행할까요?
```

**원칙**: 내용 보완 요청 시, AI 기존 지식만으로 수정하는 것을 금지한다. 반드시 search-engine 스킬로 최신 정보를 확인한 후 수정.

**내용 보완 vs 편집 구분**:
- 내용 보완: "더 자세히", "데이터 추가", "근거 보강", "최신 자료로", "트렌드 반영" → 리서치 필수
- 편집: 오타, 표현 변경, 순서 교체, 디자인/레이아웃, 이미지 교체 → 리서치 불필요

---

## §5. 리서치 면제 조건

| 조건 | 사유 |
|------|------|
| 사용자가 "리서치 없이 진행" 명시 | 사용자 의사 존중 |
| 텍스트 편집만 요청 (오타, 표현, 순서) | 내용 보완이 아닌 편집 |
| 이미지/디자인/레이아웃만 변경 | 소스 자료와 무관 |
| 해당 섹션 Source 필드에 이미 3개+ 출처 존재 | 충분한 소스 보유 |
| Step 0에서 소스 선택 6번(AI 조사) 선택 | 이미 AI 지식 기반으로 동의 |
