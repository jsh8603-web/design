## ★추가개선 2건 (개선안 위 정교화 — 핸드오프 그대로면 놓쳤을 것)

### 1. VP-04 텍스트 밝기 게이트 (validate-pptx.js checkContrast)
- **발견**: 핸드오프 "picture 겹치면 무조건 제외"는 운영덱 **소멸 34건 중 33건이 FN**. s3 청록 `$1,583억`(luma143), s90 주황 "COVID 가속"(144), 회색 출처(161) — 연한 장식 picture 위 유채색/회색 저대비 텍스트까지 과잉제외. 밝은 흰글자 진짜 FP는 1건뿐. media 밝기 측정(s3 평균153)으로 확인.
- **개선**: 도형의 모든 텍스트 run 이 밝을(luma>200) 때만 picture 제외. `litRuns.every(luma>200) AND overlapsPicture`.
- **검증**: 776→(무조건)742→(게이트)775 = 밝은글자 1건만 제거, 33 FN 복원. 합성 slide1 흰/연회 FP제거 유지 + slide2 invisible TP 유지.

### 2. PF-65 blockKids 게이트 (preflight-html.js)
- **발견**: 핸드오프 Range 측정이 복합셀(원형배지+라벨 세로배치, s4007 "+12%/긍정적")의 도형높이를 합산해 2줄로 오판 = **새 FP 생성**(카운트가 33↔33로 우연히 동일했던 이유).
- **개선**: 셀에 블록자식 ≥2(의도적 세로배치)면 wrap 아님 → 제외. `blockKids.length <= 1` 조건.
- **검증**: s4007 FP 제거 + s4013 무발화·s8012/8017 scrollWidth TP 유지. 운영덱 PF-65 33→23 순감소(무조건 Range였으면 33).
